### 現実世界のIT <-> 異世界の魔法 対応表

| 現実世界 | 異世界 | 役割の説明 |
| :--- | :--- | :--- |
| **開発言語** | **魔法体系 / 術式言語** | 魔素を制御するための「文法」と「語彙」。これがないと魔力はただ暴発する。 |
| **構文 (Syntax)** | **呪文の構成 / 術式の構造** | 言葉やシンボルの並べ方。ルールに沿わないと「構文エラー」で不発になる。 |
| **標準ライブラリ** | **教本に載る定型魔法** | 誰でも呼び出せるように最適化された、実績のある術式セット。 |
| **ランタイム / 実行環境** | **世界の物理法則 / 精霊** | コードを解釈して物理現象へと翻訳・出力する基盤。 |
| **コンパイラ** | **精神の変換機構 / 媒介** | 人間の思考（ソースコード）を、世界が理解できる波形（機械語）に変換するプロセス。 |
| **API** | **契約 / 召喚契約** | 外部の力（精霊等）を、内部ロジックを知らなくても呼び出して利用するための窓口。 |

### 魔法体系 <-> 開発言語 比較表
e.g.
| 魔法の体系 | 開発言語 | IT的な役割・立ち位置 |
| :--- | :--- | :--- |
| **精霊魔法** | **Python / Ruby** | **スクリプト系**。精霊（実行環境）に丸投げして、少ない命令で動かす「高水準」な魔法。 |
| **現代魔術** | **Java / C#** | **オブジェクト指向系**。論理的・体系的に整理されており、誰が使っても同じ結果が出る「汎用」魔法。 |
| **古代魔法** | **C / C++** | **システム記述系**。世界の物理定数や魔素を直接操作する、非常に強力だが制御が難しい「低水準」魔法。 |
| **神聖魔法** | **SQL / 宣言型言語** | **宣言型**。「何をしたいか」だけを神（システム）に伝え、その結果を得る。内部処理（HOW）は術者には見えない。 |
| **数理魔術** | **Haskell / Lisp** | **関数型**。厳密な数式や論理式によって構成され、バグ（暴走）が理論上起こらない極めて純粋な魔法。 |
| **暗黒魔法** | **JavaScript** | **動的型付け**。自由度が高く、本来なら禁止されている処理（禁忌）も強引に通せるが、カオスになりやすい。 |


### 魔法体系とITの役割 比較表

| 体系名 | 一般的なイメージ | 主な効果 | IT的な役割 |
| :--- | :--- | :--- | :--- |
| **元素魔法** | 四大属性（火水風土）、五行（木火土金水） | 攻撃、物理現象の操作 | **汎用プログラム**：自前のリソースでロジックを動かす基本形。 |
| **精霊魔法** | 精霊との契約 | 属性操作の強化、自然干渉 | **クラウド/API接続**：外部の強力なリソースを契約（認証）経由で利用。 |
| **聖教魔法** | 教会、祈り、光 | 回復、結界、魔除け | **システム管理・復旧**：システムの正常化、防御、ログ浄化。 |
| **暗黒魔法** | 呪い、代償、闇 | 弱体化、精神攻撃、死霊 | **ハッキング・ウイルス**：脆弱性を突き、相手のシステムを阻害・乗っ取り。 |
| **古代魔法** | 失われた技術、高威力 | 時空操作、広域破壊 | **低レイヤー/カーネル操作**：世界の根本（OS基盤）に干渉する特権操作。 |
